# PRD — 서울 핫플 100

- 작성일: 2026-07-09
- 상태: 초안 (v0.1)
- 플랫폼: 토스 앱인토스 (Granite React Native 미니앱)

## 1. 한 줄 정의

**"지금 서울에서 사람이 가장 많이 몰린 곳"을 실시간 TOP 100 랭킹으로 보여주는 미니앱.**

## 2. 문제와 기회

- 주말·저녁 약속 장소를 정할 때 "지금 거기 붐비나?"를 확인할 방법이 마땅치 않다. 지도 앱의 혼잡 정보는 장소 단위로 흩어져 있고, "서울 전체에서 지금 어디가 제일 핫한가"를 한눈에 보여주는 서비스는 없다.
- 서울시가 무료 공개하는 실시간 도시데이터(citydata_ppltn)는 121개 핫스팟의 혼잡도·추정 인구를 5분 주기로 제공한다. 이 데이터를 "랭킹"이라는 직관적 형식으로 재가공하면 낮은 비용으로 차별화된 콘텐츠가 된다.
- 토스 앱인토스는 유통(토스 트래픽)과 수익화(토스 애즈)를 플랫폼이 제공하므로, 콘텐츠만 좋으면 마케팅·정산 인프라 없이 수익화가 가능하다.

## 3. 타겟 사용자

| 페르소나 | 니즈 | 사용 시나리오 |
|---|---|---|
| 20~30대 나들이·데이트족 | 지금 뜨는 곳 발견 | 주말 오후 "오늘 어디 가지?" → HOT 100 상위권 탐색 |
| 약속 앞둔 직장인 | 혼잡 회피 | 퇴근 후 약속 장소 혼잡도 미리 확인 |
| 러너·산책족 | 한적한 공원 찾기 | 공원 카테고리에서 "여유" 장소 확인 |

## 4. 핵심 가치

1. **발견**: 서울 전체를 한 화면에서 — 지금 가장 붐비는 곳 TOP 100
2. **확인**: 가기 전에 혼잡도·추정 인구·12시간 예측까지 확인
3. **라이브함**: 30분마다 순위가 바뀌는 실시간성 (순위 변동 ▲▼ 표시)

## 5. 기능 정의 (v1)

### F1. HOT 100 랭킹 리스트 (홈)

- 121개 수집 장소 중 추정 인구(AREA_PPLTN_MAX, 동률 시 MIN) 내림차순 상위 100곳 표시
- 행 구성: 순위 · 순위 변동(▲n / ▼n / NEW / –) · 장소명 · 혼잡도 배지(4단계) · 추정 인구("약 2.4만~2.6만명")
- 헤더: 서비스 타이틀 + 기준 시각("15:40 기준") + 당겨서 새로고침
- 순위 변동은 직전 스냅샷(약 30분 전)과 비교해 파이프라인이 계산 — 클라이언트/DB 부담 없음

### F2. 카테고리 필터

- 칩 필터: 전체 / 관광특구 / 고궁·문화유산 / 인구밀집지역 / 발달상권 / 공원 (서울시 공식 5분류)
- 필터 적용 시에도 전체 기준 순위 번호 유지 (예: 공원 필터에서 "17위 여의도한강공원")

### F3. 장소 상세

- 혼잡도 단계 + 서울시 공식 안내 메시지(AREA_CONGEST_MSG)
- 추정 인구 범위, 데이터 기준 시각(PPLTN_TIME)
- 연령대·성별 분포 (citydata_ppltn 포함 필드, 추가 수집 비용 없음)
- 향후 12시간 혼잡 예측(FCST_PPLTN) 타임라인
- 예측 미제공(FCST_YN=N) 시 예측 섹션 숨김

### F4. 인앱 광고 (수익화)

- 배너(InlineAd) 3지면: 홈 리스트 5위 아래 1개, 홈 리스트 최하단 1개, 상세 화면 하단 1개
- 토스앱 5.241.0 미만은 광고 API 미지원 → 버전 가드 후 광고 영역 미노출 (빈 화면 방지)
- 전면형·보상형 광고는 v1 범위 외 (§9)

### F5. 데이터 상태 처리

- 로딩: 스켈레톤 UI
- fetch 실패: 오류 안내 + 재시도 버튼
- 데이터 지연: `generatedAt`이 현재보다 30분 이상 과거 → "데이터가 잠시 지연되고 있어요" 배지
- 새벽 01~05시(KST): 서울시 API 미제공 시간대 → "새벽 시간에는 실시간 데이터가 제공되지 않아요" 안내 + 마지막 데이터 표시

## 6. 비기능 요건

| 항목 | 요건 |
|---|---|
| 초기 로딩 | 데이터 JSON 1회 fetch (200~500KB), 3G 기준 3초 내 첫 렌더 목표 |
| 데이터 신선도 | 30분 주기 갱신 (01~05시 KST 제외) |
| 서버 비용 | 0원 (GitHub Actions + GitHub Pages 정적 호스팅) |
| 번들 크기 | 앱인토스 정책: 압축 해제 기준 100MB 이하 (실제 목표 « 10MB) |
| CORS | 정적 호스트가 `https://<appName>.apps.tossmini.com` origin 허용 필요 (GitHub Pages는 `ACAO: *`로 충족) |
| 데이터 고지 | "인구는 KT·SKT 통신 신호 기반 추계치" 및 출처(서울 열린데이터광장) 명시 — 상세 화면 하단 |

## 7. 수익 모델

- **v1: 토스 애즈 배너 광고** (토스 애즈 + 구글 애드몹 통합 SDK, 자동 미디에이션)
  - 수익 공식: 노출 수 × eCPM ÷ 1,000
  - 3지면 × 세션당 평균 노출로 광고 인벤토리 확보. 결제 기능 불필요 → 출시 첫날부터 수익 발생 가능
- **v2 후보: 보상형 광고** — "혼잡 추이 그래프" 등 프리미엄 콘텐츠를 광고 시청으로 잠금 해제 (eCPM이 배너 대비 높음)
- IAP(광고 제거)는 파트너 서버 + mTLS가 필요해 서버리스 원칙과 충돌 → 채택 안 함
