# 디자인 가이드 — 서울 핫플 HOT 100

- 작성일: 2026-07-09
- 상태: 초안 (v0.1)
- 선행 문서: [PRD.md](./PRD.md), [architecture.md](./architecture.md)

## 1. 디자인 원칙

1. **토스 네이티브 룩**: 레이아웃·타이포·인터랙션은 TDS React Native 컴포넌트를 그대로 사용한다. 커스텀 스타일은 혼잡도 컬러 시스템에만 한정한다.
2. **랭킹이 주인공**: 홈 화면의 시각적 위계는 순위 숫자 > 장소명 > 혼잡도 배지 > 인구 순.
3. **한 손 사용**: 필터 칩·리스트 모두 세로 스크롤 하나로 조작. 가로 스크롤은 필터 칩만 허용.
4. **광고는 콘텐츠 흐름 안에**: 배너는 리스트 행과 동일한 여백 리듬으로 삽입, 로드 실패 시 높이 0으로 접혀 빈 자리를 남기지 않는다.

## 2. 혼잡도 컬러 시스템

서비스 고유의 유일한 커스텀 시스템. 최종 색상값은 구현 시 TDS 시맨틱 토큰에서 가장 가까운 값을 채택한다 (아래 hex 는 방향 제시용).

| 단계 | 시맨틱 | 배지 색 (방향) | 라벨 |
|---|---|---|---|
| 여유 | positive | green 계열 (#1FC47D 방향) | 여유 |
| 보통 | neutral-active | blue 계열 (#3182F6 방향) | 보통 |
| 약간 붐빔 | warning | orange 계열 (#FF9E1B 방향) | 약간 붐빔 |
| 붐빔 | danger | red 계열 (#F04452 방향) | 붐빔 |

- 배지는 옅은 배경 + 진한 텍스트 조합 (TDS Badge 스타일 준수)
- 색만으로 단계를 구분하지 않는다 — 배지에 항상 텍스트 라벨 병기 (접근성)

## 3. 화면 명세

### 3.1 홈 — HOT 100 랭킹

```
┌──────────────────────────────────┐
│ 서울 핫플 HOT 100                 │  ← 타이틀 (TDS Top 영역)
│ 15:40 기준 · 10분마다 업데이트     │  ← 캡션 (지연 시 "⚠ 데이터 지연" 배지 추가)
├──────────────────────────────────┤
│ [전체] [관광특구] [발달상권] [공원]…│  ← 카테고리 칩 (가로 스크롤)
├──────────────────────────────────┤
│ 1  ▲2  강남역                     │
│        [붐빔]  약 2.4만~2.6만명    │
│ 2  –   홍대 관광특구               │
│        [붐빔]  약 2.2만~2.4만명    │
│ 3  ▼1  성수카페거리               │
│        [약간 붐빔] 약 1.8만~2.0만명 │
│ 4  NEW 여의도한강공원              │
│ 5  …                             │
│ ┌──────────────────────────────┐ │
│ │        배너 광고 (InlineAd)    │ │  ← 5위 아래
│ └──────────────────────────────┘ │
│ 6  …                             │
│    (계속)                        │
│ 100 …                            │
│ ┌──────────────────────────────┐ │
│ │        배너 광고 (InlineAd)    │ │  ← 최하단
│ └──────────────────────────────┘ │
│ 출처: 서울 열린데이터광장           │  ← 푸터 캡션
└──────────────────────────────────┘
```

- **랭킹 행 (RankingRow)**: 순위 숫자는 1~3위 강조(굵게+크게), 4위 이하 일반. 순위 변동은 ▲(빨강 아님 — 상승은 긍정이므로 red 지양, TDS 기본 강조색)·▼(회색)·NEW(포인트색) 캡션
- **당겨서 새로고침**: RefreshControl, 완료 시 기준 시각 갱신
- **필터 칩**: 선택 시 즉시 리스트 필터링, 순위 번호는 전체 기준 유지
- **행 탭** → 상세 화면 push

### 3.2 상세 — 장소 상세

```
┌──────────────────────────────────┐
│ ←  강남역                         │  ← 내비게이션 바
├──────────────────────────────────┤
│ 지금 1위 · 발달상권                │
│                                  │
│        [ 붐빔 ]                   │  ← 대형 혼잡도 배지
│  "사람이 몰려있을 수 있어요"        │  ← 서울시 공식 메시지
│  약 24,000~26,000명 · 15:30 기준  │
├──────────────────────────────────┤
│ 누가 많을까요?                     │
│  20대 28% · 30대 22% · 40대 16% … │  ← 연령대 상위 3~4개 + 성별 비율 바
├──────────────────────────────────┤
│ 앞으로 12시간 예측                 │
│  16시 [붐빔] 17시 [붐빔] 18시 [약간붐빔]…│  ← 가로 스크롤 타임라인 칩
├──────────────────────────────────┤
│ ┌──────────────────────────────┐ │
│ │        배너 광고 (InlineAd)    │ │
│ └──────────────────────────────┘ │
│ ⓘ 인구는 KT·SKT 통신 신호 기반     │
│   추계치예요. 출처: 서울 열린데이터광장│
└──────────────────────────────────┘
```

- 예측 미제공(forecast 빈 배열) 시 예측 섹션 전체 숨김
- 연령대 분포는 그래프 라이브러리 없이 TDS ProgressBar/바 형태로 단순 표현

### 3.3 상태 화면

| 상태 | UI |
|---|---|
| 로딩 | 리스트 스켈레톤 (행 8개 + 칩 자리) |
| 오류 | 일러스트 없이 텍스트 + "다시 시도" 버튼 (TDS Button) |
| 지연(stale) | 정상 리스트 위 기준시각 옆 경고 배지 |
| 새벽(night) | 리스트 상단 안내 배너: "새벽 1~5시에는 실시간 데이터가 제공되지 않아요. 마지막 데이터를 보여드릴게요" |

## 4. TDS 컴포넌트 매핑

| UI 요소 | TDS RN 컴포넌트 (구현 시 정확한 이름 확인) |
|---|---|
| 타이틀·캡션 | Text (타이포 토큰) |
| 카테고리 칩 | Chip / SegmentedControl 계열 |
| 혼잡도 배지 | Badge |
| 리스트 행 | ListRow / List 계열 |
| 새로고침 | RN RefreshControl |
| 버튼 | Button |
| 분포 바 | ProgressBar |
| 안내 배너 | Banner / Callout 계열 |

- 구현 단계에서 `tds-rn` 문서 검색(docs-search 스킬)으로 실제 컴포넌트명·prop 확정
- 로컬 브라우저에서 TDS 미동작 → 샌드박스 앱에서 시각 확인

## 5. 광고 배치 규칙

1. 홈 리스트 5위와 6위 사이 — 첫 화면 스크롤 직후 자연 노출
2. 홈 리스트 100위 아래 — 완주 사용자 대상
3. 상세 화면 콘텐츠 아래, 출처 고지 위
- 첫 뷰포트(스크롤 전 화면)에는 광고를 두지 않는다 — 콘텐츠 신뢰 우선
- 광고 라벨·스타일은 SDK 기본값 유지 (심사 리스크 최소화)

## 6. 앱 아이덴티티

- `primaryColor`: 토스 블루(#3182F6) 기본 유지 — 혼잡도 컬러와 충돌하지 않는 중립 아이덴티티
- 앱 아이콘: 서울 스카이라인 또는 지도핀 + 순위 모티프 (콘솔 등록 시 제작, 오픈 이슈)
- 서비스명: PRD 오픈 이슈 1 (콘솔 등록 전 확정)
